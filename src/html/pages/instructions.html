How to use Calcium in your project

Include Calcium.

Include the production files from the gh-pages branch. We recommend either downloading the releases or using a submodule.

Calcium CSS.

For simple projects, you can just load the compiled CSS files directly in your page.

<link rel="stylesheet" href="calcium/assets/styles/calcium.css" />

Calcium SASS.

For more complex projects, you can include the SCSS files at the top of your SCSS file. You will need to define the project path so that images load correctly. See the example SCSS file below.

// Define image path, relative to your project's main SASS file.
$cd-images: "./../vendor/calcium/assets/images";

// Include the Calcium SASS file.
@import "./../vendor/calcium/assets/sass/calcium";

// Your custom styles here.

JavaScript.

Calcium aims to have minimal JavaScript, and it is completely optional. The small ammount there is should be as lightweight as possible, without any dependencies. You can include the calcium.min.js file in your project for a few nice little extra features (e.g. code syntax highlighting).

<script src="calcium/assets/js/calcium.min.js"></script>

Fonts.

Calcium uses IBM Plex Mono for both headings and body fonts. You will need both 200 and 600 weights. These fonts are open source and available to use as web fonts, or you can load them via Typekit or Google Fonts if preferred.

Updates.

We aim to maintain full compatability between minor releases however major releases may include breaking changes.
